Que es GitFlow
Git es el sistema de control de versiones más utilizado del mundo. Este es un proyecto de código abierto y con mantenimiento activo desarrollado principalmente por Linus Torvalds, el creador del kernel de Linux. Su diseño fue basado en BitKeeper y en Monotone.

Git Flow es un modelo alternativo de creación de ramas Git publicado por primera vez y popularizado por Vincent Driessen en nvie en la que se utilizan ramas de función y varias ramas principales. GitFlow puede utilizarse en proyectos de entrega continua.
En vez de gastar una única rama este utiliza dos, “main” y “develop”, main es la que se encarga de almacenar el historial de publicaciones oficial y la rama develop es la que se encarga como rama de integración para las funciones.


Las ramas de función utilizan la rama develop como rama primaria y de ahí cuando una función está terminada se vuelve a fusionar con develop. Las funciones no deben interactuar nunca directamente con la rama main. Cuando la rama de develop haya tenido suficientes funciones como para una publicación se crea una rama “release” a partir de develop. Una vez creada esta rama no se pueden añadir nuevas funciones, solamente soluciones de errores, generación de documentación u otras tareas orientadas a la publicación de esta. Cuando la rama “release” esté lista esta se fusiona con la rama main y se etiqueta con el número de versión. La rama de corrección o “hotfix” sirve para reparar rápidamente las publicaciones de producción. La rama “hotfix” es muy similar a las ramas “release” y “feature” solametnte que esta actua sobre la rama “main” y no sobre la rama “develop”. En general, gitflow es ideal para proyectos basados en publicaciones y sobretodo ideal por su canal para las correciones de producción, “hotflix”
